2. SEVERITY BUG CLASSIFICATION
────────────────────────

Каждый баг классифицируй:

P0 — критично:

* невозможно зарегистрироваться/войти;
* приложение падает;
* белый экран;
* потеря данных;
* доступ к чужим данным;
* обход ролей;
* не работает создание AI-агента;
* не работает основной flow лида/сообщения/звонка;
* платежи ломаются;
* security issue.

P1 — серьёзно:

* ключевая кнопка не работает;
* форма не сохраняется;
* неправильная навигация;
* важная страница ломается на mobile;
* analytics показывает неверно;
* CRM не создаёт лид;
* inbox не обновляется;
* ошибки API;
* плохой UX в главном сценарии.

P2 — средне:

* визуальный баг;
* мелкая UX-проблема;
* непонятный текст;
* нет loading state;
* нет empty state;
* плохой hover/focus;
* неконсистентные отступы;
* неидеальная адаптивность.

P3 — улучшение:

* можно улучшить текст;
* можно повысить конверсию;
* можно сделать интерфейс понятнее;
* можно добавить подсказку;
* можно улучшить onboarding.

Для каждого бага указывай:

* ID;
* severity;
* страница;
* устройство/размер экрана;
* браузер;
* шаги воспроизведения;
* фактический результат;
* ожидаемый результат;
* скриншот/описание;
* console errors;
* network errors;
* предполагаемая причина;
* рекомендация по исправлению.

────────────────────────

75–90% — production-ready для малого бизнеса.
